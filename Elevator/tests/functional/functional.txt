预计时间：开关门各1s,电梯运行2s/楼，开门后等待时间3s

调度算法：
TestCase1:
1. 初始时，两台电梯都在一层,乘客A按下一楼电梯外部按钮up, 进入电梯后按下内部按钮3
2. 在电梯1向上运行1s时，乘客B按下二楼的外部按钮up
3. 电梯停靠二楼，乘客B进入电梯
4. 电梯到达三楼，乘客A、B到达目标楼层

TestCase2:
1. 初始时，两台电梯都在一层,乘客A按下一楼电梯外部按钮up, 进入电梯后按下内部按钮3
2. 在电梯1向上运行1.8s时，乘客B按下二楼的外部按钮up
3. 由于电梯1临近二楼无法减速，电梯1继续向上运行
4. 电梯2向上运行至二楼，乘客B进入，按下内部按钮3
4. 电梯1、2先后到达三楼，乘客A、B到达目标楼层

TestCase3:
1. 初始时，两台电梯都在一层，位于2层的乘客A和B间隔0.5s时间分别按下位于二楼的外部按钮up 和 down
2. 两部电梯运行至二楼，等待电梯门关闭后，位于二楼的乘客B和A再次间隔0.5s时间分别按下位于二楼的外部按钮down和 up 
3. 乘客B进入电梯2，按下电梯内部按钮-1, 然后乘客A进入电梯1，按下电梯内部按钮3
4. 电梯2到达-1层，电梯1到达3层，乘客A、B到达目标楼层

TestCase4:
1. 位于电梯1内部的A和位于电梯内部的B间隔0.5s分别按下内部按钮3，两部电梯同时向上运行
2. 位于二层的乘客C，按下二楼的外部按钮up
3. 先出发的电梯1停靠三层后，继续前往二层，乘客C进入电梯
4. 电梯2继续停止在三层

TestCase5:
1. 位于电梯2内部的B按下内部按钮-1，电梯2向下运行至-1层
2. 电梯1内部的A按下内部按钮3，电梯1运行至3层
3. 乘客A按下电梯内部按钮-1，乘客B按下电梯内部按钮3，电梯开始运行
4. 位于一层的乘客D，按下一楼的外部按钮down，位于二层的乘客C，按下二楼的外部按钮up
5. 电梯1停靠一层，乘客D进入
6. 电梯2停靠二层，乘客C进入
7. 电梯1到达-1层，电梯2到达三层，乘客A、B、C、D到达目标楼层

TestCase6:
1.位于电梯1内部的乘客A按下内部按钮3，电梯向上运行
2.在电梯1经过二层之后，乘客A按下内部按钮2，无反应
3.电梯1继续运行到达三楼

TestCase7:
1.位于电梯1内部的乘客A按下内部按钮3，电梯向上运行
2.在电梯1运行1.5s后，乘客A按下内部按钮2，由于电梯1临近二层无法停靠，无反应
3.电梯1继续运行到达三楼

TestCase8:
1.位于电梯1内部的乘客A按下内部按钮3，电梯向上运行
2.在电梯1运行1s后，乘客A按下内部按钮2，二层灯亮起
3.电梯1停靠二楼，之后电梯1继续运行到达三楼

门控系统：
TestCase1:
1. 位于一楼的乘客A按下外部按钮up,电梯1门打开，乘客A进入按下内部按钮2
2. 在门将关闭时，点击开门按钮，门打开 
3. 等待门再次关闭，电梯1上行，乘客A到达二楼

TestCase2:
1. 位于一楼的乘客A按下外部按钮up,电梯1门打开，乘客A进入按下内部按钮2
2. 在门在打开等待时，连续点击开门按钮3次，门持续打开 
3. 等待门再次关闭，电梯1上行，乘客A到达二楼

TestCase3:
1. 位于一楼的乘客A按下外部按钮up,电梯1门打开，乘客A进入按下内部按钮2
2. 在门将打开时时，点击关门按钮，门继续打开 
3. 等待门再次关闭，电梯1上行，乘客A到达二楼

TestCase4:
1. 位于电梯1内部的A和位于电梯内部的B间隔0.5s分别按下内部按钮3，两部电梯同时向上运行
2. 位于二层的乘客C，按下二楼的外部按钮up
3. 先出发的电梯1停靠三层后, 乘客A持续点击开门按钮3次
4. 电梯2继续前往二层，乘客C进入电梯
